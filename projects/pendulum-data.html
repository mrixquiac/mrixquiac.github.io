Análisis de datos experimentales del péndulo

Procesamiento de datos reales de un péndulo físico, ajuste de parámetros por mínimos cuadrados y comparación cuantitativa con la solución numérica del modelo teórico.

El problema

El péndulo simple es uno de los sistemas más estudiados en física, pero en el laboratorio nunca se comporta exactamente como el modelo ideal. Hay amortiguamiento, el eje no es perfectamente rígido, el ángulo inicial puede ser suficientemente grande para que la aproximación de ángulo pequeño deje de ser válida. El objetivo de este proyecto fue tomar datos reales, procesarlos con cuidado y cuantificar qué tan bien los explica el modelo teórico.

Las preguntas concretas que quería responder eran: ¿cuánto amortiguamiento tiene el sistema? ¿Es consistente con fricción viscosa o con fricción de Coulomb? ¿El período experimental coincide con la predicción teórica para ángulos grandes?

Método

Los datos se obtuvieron con un sensor de posición angular a 200 Hz durante 60 segundos. El primer paso fue limpiar la señal: eliminar outliers, corregir el offset de cero y suavizar el ruido de alta frecuencia con un filtro de Savitzky-Golay que preserva la forma de los picos.

Lectura y limpieza de datos

import pandas as pd
import numpy as np
from scipy.signal import savgol_filter

# Leer datos del sensor (columnas: tiempo [s], ángulo [rad])
df = pd.read_csv('pendulum_raw.csv', names=['t', 'theta'])

# Eliminar outliers: puntos a más de 3σ de la media local
rolling_mean = df['theta'].rolling(window=20, center=True).mean()
rolling_std  = df['theta'].rolling(window=20, center=True).std()
mask = np.abs(df['theta'] - rolling_mean) < 3 * rolling_std
df   = df[mask].reset_index(drop=True)

# Suavizado Savitzky-Golay (preserva picos y valles)
df['theta_smooth'] = savgol_filter(df['theta'], window_length=11, polyorder=3)

Ajuste por mínimos cuadrados

El modelo teórico para el péndulo amortiguado con ángulos pequeños es una oscilación sinusoidal con envolvente exponencial decreciente. Utilicé scipy.optimize.curve_fit para ajustar los cuatro parámetros del modelo a los datos suavizados. 

from scipy.optimize import curve_fit

def pendulo_amortiguado(t, A, gamma, omega, phi):
    """Oscilador armónico amortiguado subcríticamente."""
    return A * np.exp(-gamma * t) * np.cos(omega * t + phi)

# Estimación inicial de parámetros
theta0 = df['theta_smooth'].max()
p0 = [theta0, 0.05, 2*np.pi/2.1, 0.0]  # A, γ, ω, φ

popt, pcov = curve_fit(
    pendulo_amortiguado,
    df['t'],
    df['theta_smooth'],
    p0=p0,
    maxfev=5000
)

A, gamma, omega, phi = popt
perr = np.sqrt(np.diag(pcov))  # incertidumbres (1σ)

print(f"γ = {gamma:.4f} ± {perr[1]:.4f} s⁻¹")
print(f"ω = {omega:.4f} ± {perr[2]:.4f} rad/s")
print(f"T = {2*np.pi/omega:.4f} s")

Resultados

El ajuste converge con residuos cuadráticos medios de 0.003 rad, lo que representa un error relativo menor al 2% respecto a la amplitud inicial. El período experimental resultó T = 2.134 ± 0.003 s, consistente con la predicción teórica T = 2π√(L/g) = 2.128 s para la longitud medida del péndulo.

El coeficiente de amortiguamiento γ = 0.043 s⁻¹ es coherente con fricción viscosa en el eje, no con fricción de Coulomb (que habría producido una envolvente lineal, no exponencial). La comparación entre ambos modelos mediante el criterio de información de Akaike confirma que el modelo viscoso es preferible.

La concordancia entre el período experimental y la predicción teórica al nivel de 0.3% sugiere que la aproximación de ángulo pequeño es válida para el rango de ángulos iniciales usado (θ₀ ≈ 15°).

Lo que aprendí

La parte más instructiva fue la limpieza de datos, no el ajuste. Los outliers del sensor afectan el ajuste de forma desproporcionada porque curve_fit minimiza la suma de errores al cuadrado — un único punto muy alejado puede dominar el resultado completo. El filtro de Savitzky-Golay fue la solución correcta porque preserva los extremos de la oscilación, que son los puntos más informativos para estimar la amplitud y el amortiguamiento.

También aprendí a reportar incertidumbres correctamente. La matriz de covarianza de curve_fit da las incertidumbres estadísticas del ajuste, pero no incluye las incertidumbres sistemáticas del instrumento. Ignorar esa distinción lleva a reportar precisiones irrealmente altas.
